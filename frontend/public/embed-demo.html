HackathOERn Ideendatenbank — Integration in Drittseiten

Die App stellt zwei Web Components bereit, die sich per HTML-Tag in beliebige Seiten (WordPress, statische Seiten, andere CMS) einbetten lassen. Unten drei typische Einbett-Szenarien mit Live-Rendering.

Setup: einmalig <script src="main.js"> und <link rel="stylesheet" href="styles.css"> in den Seitenkopf. Danach können die Tags beliebig oft verwendet werden.

1. Komplette Ideen-App

Nimmt die ganze verfügbare Seitenbreite und bringt Navigation, Filter, Einreichungsformular und Detailseiten mit. Typisch für eine eigene „Ideendatenbank"-Unterseite im CMS.

<ideendb-app api-base="https://api.example.com/api/v1"></ideendb-app>

2. Kachel-Grid für die Startseite

Zeigt die neuesten Ideen als Teaser — ideal für Blog-Startseiten oder eine „Aktuelle Ideen"-Box in der Sidebar.

Neueste Ideen (Limit 6)

<ideendb-tile-grid
  api-base="https://api.example.com/api/v1"
  sort="modified"
  limit="6">
</ideendb-tile-grid>

3. Top-Ideen eines HackathOERn-Events

Zeigt nur Ideen eines bestimmten Events — praktisch für Event-Landingpages („Unsere Ideen beim HackathOERn 2"). Das event-Attribut nimmt den Slug aus der Event-Taxonomie (klein, mit Bindestrichen).

Ideen beim HackathOERn 2

<ideendb-tile-grid
  api-base="https://api.example.com/api/v1"
  event="hackathoern-2"
  sort="rating"
  limit="4">
</ideendb-tile-grid>

4. Ideen einer Herausforderung

Filter nach Topic-ID (Sammlungs-ID aus edu-sharing) — für Landingpages, die zu einer konkreten Herausforderung gehören.

Auffindbarkeit und Nutzungsverbesserung von OER

<ideendb-tile-grid
  api-base="https://api.example.com/api/v1"
  topic-id="430ebc36-30c7-4ce0-8ebc-3630c7ece08c"
  limit="6">
</ideendb-tile-grid>

Attribute-Übersicht

<ideendb-app>

  • api-base — URL des FastAPI-Backends (Default: http://127.0.0.1:8000/api/v1)

<ideendb-tile-grid>

  • api-base — Backend-URL
  • topic-id — edu-sharing Sammlungs-ID (alle Ideen aus Sub-Sammlungen werden mitgezogen)
  • phase — Phase-Filter (z.B. „Anregung", „In Umsetzung")
  • event — Event-Filter (z.B. „HackathOERn 2")
  • q — Freitextsuche
  • sort — modified | created | rating | comments | title (Default: modified)
  • limit — Anzahl (Default: 12)
  • hide-footer — true blendet den „Mehr laden"-Button aus

CORS / Deployment

Das FastAPI-Backend erlaubt per Default lokale Dev-Hosts (localhost, 127.0.0.1). Für Produktions-Einbettung setze APP_CORS_ORIGINS im Backend-Env auf deine CMS-Domain:

APP_CORS_ORIGINS=https://www.wirlernenonline.de,https://hackathoern.openeduhub.net
